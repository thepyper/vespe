# Plan for Adding an Executable Header

This plan outlines the steps to add a decorative header to the console output when the Vespe executable starts, displaying version and copyright information.

## 1. Identify Header Content

- **Application Name:** Vespe
- **Version:** Retrieved dynamically from `Cargo.toml` (`env!("CARGO_PKG_VERSION")`)
- **Copyright:** Copyright ThePyper

## 2. Locate Startup Point

- The `run()` function in `src/lib.rs` is the primary application entry point and the ideal place to print the header.

## 3. Implement Header Printing

- **File to modify:** `src/lib.rs`
- **Action:** Add `println!` statements at the beginning of the `run()` function.
- **Details:**
    - The header will include the application name, version, and copyright.
    - Decorative lines will be added for visual appeal.

## 4. Verification

- **Action:** Build the project using `cargo build`.
- **Action:** Run the application (e.g., `cargo run -- chat my_agent "Hello"`).
- **Verification:** Observe the console output to ensure the header is printed correctly at startup.
