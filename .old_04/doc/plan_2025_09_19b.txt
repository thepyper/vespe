# Piano di Lavoro: Creazione Dataset per `buzz`

Data: 2025-09-19

## 1. Obiettivo

L'obiettivo di questo piano è documentare la procedura per la creazione di un dataset di addestramento per il progetto `buzz`. Questo dataset verrà usato per addestrare un modello di sequence labeling capace di segmentare l'output di un LLM in categorie funzionali.

## 2. Richiesta Iniziale

L'utente ha richiesto di creare un ambiente di training e un set di dati di esempio. La richiesta specificava:

1.  La creazione di una struttura di directory: `buzz/training/2025_09_19_gemini`.
2.  La generazione di file di esempio all'interno di questa directory.
3.  L'adozione di un formato specifico per questi file.

## 3. Formato dei Dati Sorgente

Abbiamo concordato di usare un formato testuale intermedio, facile da leggere e scrivere sia per umani che per LLM. Questo formato servirà come "sorgente" per i dati, da convertire successivamente nel formato JSONL richiesto dal framework di training.

**Specifiche del formato:**

-   **Un file per ogni response:** Ogni file `.txt` rappresenta una singola risposta completa dell'agente.
-   **Un segmento per riga:** Ogni riga del file contiene un solo segmento funzionale.
-   **Label esplicita:** Ogni riga deve iniziare con una delle seguenti etichette, seguita da due punti e uno spazio: `LABEL: `.

**Etichette definite:**

-   `TEXT`: Testo libero destinato all'utente finale.
-   `THOUGHT`: Ragionamenti interni del modello.
-   `TOOL_CALL`: Una chiamata a un tool, che può essere in qualsiasi formato (XML, JSON, testo semplice, ecc.).
-   `TOOL_RESPONSE`: L'output restituito da un tool.

**Esempio:**

```
THOUGHT: Devo leggere il file `Cargo.toml` per vedere le dipendenze.
TOOL_CALL: <tool_code>read_file('Cargo.toml')</tool_code>
TOOL_RESPONSE: <tool_response status='success'>[dependencies]\ntokio = "1"</tool_response>
THOUGHT: Ok, ora posso rispondere all'utente.
TEXT: Il progetto usa `tokio` come dipendenza.
```

## 4. Piano di Generazione

1.  È stata creata la directory richiesta.
2.  Sono stati generati 10 file di esempio iniziali per validare il formato e la varietà.
3.  Su richiesta dell'utente, si procederà a generare altri 200 esempi, mantenendo un'elevata varietà di scenari, formati e sequenze per garantire la robustezza del futuro modello.
